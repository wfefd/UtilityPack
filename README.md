## 1. 과제 소개: UtilityPack

`UtilityPack`은 팀원이 각각 구현한 유틸리티 기능 5가지를 하나의 Tkinter GUI 메뉴에서 선택해 실행할 수 있도록 만든 파이썬 프로그램입니다.

### 단위 변환기 (unit_converter_hyungkyu.py)

- 길이, 면적, 부피, 속도, 시간 등의 여러 단위를 서로 변환하는 도구  
- 카테고리별로 기본 단위(예: 길이 = m, 시간 = s)를 기준으로 변환 계수를 사용  
- 사용자가:
  - 카테고리 선택
  - 시작 단위 / 목표 단위 선택
  - 변환할 값 입력  
  → 변환 버튼 클릭 시 결과를 `"입력값 from = 결과값 to"` 형식으로 표시  
- Tkinter `OptionMenu`, `Entry`, `Label`, `Button`을 사용해 GUI로 구현

### 간단 계산기 (simple_calculator_jincheol.py)

- 두 숫자를 입력받아 사칙연산(+, -, ×, ÷)을 수행하는 간단 계산기
- 연산자 선택 후 결과를 화면에 출력
- 잘못된 입력(숫자가 아닌 값 등)에 대한 예외 처리 포함

### 점수 평균 계산 (score_average_harin.py)

- 여러 과목/시험 점수를 입력받아 평균 점수를 계산
- 공백으로 구분된 점수들을 입력 → 리스트로 변환 후 평균 계산
- 결과를 소수점 자리까지 보기 좋게 표시

### 문자열 처리 (string_tools_seunghwan.py)

- 문자열에 대해 다음과 같은 간단한 문자열 기능 제공
  - 대문자 / 소문자 변환
  - 공백 제거 또는 트리밍
  - 문자열 길이 확인 등
- 사용자가 문자열과 원하는 기능을 선택해서 실행할 수 있는 형태

### 랜덤 비밀번호 생성기 (password_generator_jngsu.py)

- 원하는 길이를 지정하면 랜덤 비밀번호를 생성해 주는 기능
- 사용 문자:
  - 영문 소문자 (a–z)
  - 영문 대문자 (A–Z)
  - 숫자 (0–9)
  - 특수문자 (`!@#$%^&*`)
- 길이를 입력하면 해당 길이의 랜덤 비밀번호를 만들어 팝업으로 보여줌

## 4. Git 협업 기록

### 4-1. 브랜치 전략

- 기본 브랜치: `main`
- 기능별 브랜치에서 작업 후 `main`에 병합하는 방식 사용
- 예시 브랜치:
  - `feature/unit-converter`
  - `feature/string-tools`
  - `feature/score-average`
  - `feature/simple-calculator`
  - `feature/password-generator`
  - 각 브랜치에서 기능 구현 후 Pull Request(PR)를 생성하여 코드 리뷰 및 병합 진행

### 4-2. Pull Request(PR) 현황(예시)

- 전체 PR 개수: **약 8개**
  - 기능 구현 PR: 5개 (각 기능별 1개씩)
  - 메인 메뉴 연동 및 UI 수정 PR: 2개
  - 버그 수정 및 리팩토링 PR: 1개
- PR 작성 시 공통 포함 내용:
  - 변경된 기능 및 파일 요약
  - 실행/테스트 방법 및 확인 사항
  - 필요 시 동작 화면 캡처 또는 설명 추가

### 4-3. Merge & Conflict 해결 경험(예시)

- 공용 파일에서 여러 차례 충돌(Conflict) 발생 경험
- 주요 충돌 파일: `main_menu.py`
  - 여러 기능을 한 메뉴에서 호출하는 과정에서, 서로 다른 브랜치에서 같은 부분(메뉴/버튼 영역)을 수정하여 충돌 발생
- 해결 절차:
  1. `git pull origin main`으로 최신 `main` 내용 반영
  2. 충돌 표시 구간(`<<<<<<<`, `=======`, `>>>>>>>`) 확인
  3. 각 기능의 버튼 및 메뉴 항목을 모두 포함하도록 수동 병합
  4. 로컬에서 프로그램 실행 후 전체 메뉴 클릭 테스트
  5. 정상 동작 확인 후 커밋 및 PR 업데이트
- 프로젝트 소개/README 관련 텍스트에서도 일부 충돌 발생
  - 팀원별로 추가한 설명을 합쳐서 자연스럽게 이어지도록 수동 수정

이를 통해 공용 파일 작업 시:

- 작업 전 항상 최신 `main`을 기준으로 브랜치를 업데이트하는 습관 형성
- 누가 어느 부분을 수정 중인지 사전에 공유하는 것이 중요하다는 점을 경험

### 4-4. 협업 통해 얻은 점(예시)

- Git 브랜치 & PR 기반 협업을 통해:
  - 각자 맡은 기능을 독립적으로 개발하면서도
  - `main` 브랜치를 비교적 안정적인 상태로 유지할 수 있었음
- 코드 리뷰 과정에서:
  - 개인이 놓친 예외 상황 및 코드 스타일 문제를 서로 보완
  - 공통된 코드 스타일과 UI 흐름을 맞추는 데 도움이 됨
- Conflict 해결 경험을 통해:
  - 단순히 “충돌 표시를 없애는 것”이 아니라  
    최종적으로 **모든 기능이 정상 동작하도록 코드를 합치는 것**이 중요하다는 점을 학습
